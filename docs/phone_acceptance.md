# 真机验收清单

## 连接手机

1. 手机设置里连续点 7 次“版本号”，打开开发者选项。
2. 在开发者选项里打开“USB 调试”。
3. 用支持数据传输的 USB 线连接 Mac。
4. 手机弹窗选择允许调试。
5. 在项目根目录运行：

```bash
adb devices -l
```

看到一台设备状态为 `device` 后继续。

小米 / HyperOS / MIUI 设备如果出现 `INSTALL_FAILED_USER_RESTRICTED`，需要在开发者选项里打开“USB 安装 / 通过 USB 安装”，并在手机弹出的安装确认里点允许。

## 自动验收

```bash
scripts/install_and_test_device.sh
```

脚本会检查：

- 只连接了一台已授权设备。
- 设备支持 `arm64-v8a`。
- `/data` 分区大致有 3 GB 以上可用空间。
- Debug APK 可以安装。
- 首屏 Compose 冒烟测试通过。
- App 可以被启动。

脚本会卸载旧调试包并清空 App 数据，用来保证首启测试是干净状态。已经下载好的模型会被清掉，做完整模型验收前请确认可以重新下载或重新导入。

## 手动模型验收

1. 打开“Gemma 随身问答”，确认首屏显示“设备检查”。
2. 可用空间足够时点击“下载推荐模型”。
3. 下载中确认能看到进度、字节数和取消入口。
4. 下载完成后确认状态变成“离线可用”。
5. 关闭 Wi-Fi/移动网络后输入一个中文问题，确认能生成回答。
6. 点右上角新会话按钮，确认聊天记录清空且仍可继续问答。
7. 如果 GPU 初始化失败，确认 App 自动切到 CPU，且仍可问答。

## 导入模型验收

1. 卸载重装或清除 App 数据。
2. 把 `gemma-4-E2B-it.litertlm` 放到手机存储。
3. 打开 App，点击“导入已有模型”。
4. 选择 `.litertlm` 文件，确认导入进度可见。
5. 导入完成后确认自动加载并可离线问答。

## 失败场景

- 用非 `.litertlm` 文件导入，应显示“请选择 .litertlm 模型文件”。
- 存储空间不足时，应显示空间不足提示。
- 断网或非 Wi-Fi 时，下载应等待或失败，并给出可理解状态。
